Application Programming Interface (API)

Scope

The API service of the FTA Portal provides third party organisations and systems access to the same data and functionality that enables the website. The API has been made available to encourage users to find new ways to use, display and share preferential tariff and rules of origin data relating to Australiaâ€™s free trade agreements.

Access, Usage, Performance and Availability

The API currently supports unlimited anonymous usage. It also includes mechanisms (e.g. data versioning) to allow user side caching and re-use of data if additional performance is needed.

The API usage and performance is being monitored closely and unlimited anonymous usage may change in the future, if necessary, such as owing to the misuse of data. Future versions of the API may require authentication and/or apply rate limits.

We aim to provide 99.95% service availability.

Future Changes

Future changes to the API will be announced on the blog.

Older versions of the API will be supported for a minimum period of three months. The results returned from any API calls will also include a deprecation message in this period with specific details.

Examples

The DFAT FTA Portal website is built entirely using this API. The following examples and associated modules, released under the MIT license, are provided as a demonstration of how to use the API.

  • Products Search - Example of how to search for keywords or HS codes and return relevant HS product categories.

  • Products Hierarchy Browser - Example of how to use and implement a hierarchy view of tariff schedule HS code categories to the four-digit level.

  • Products Data Viewer - Example of how to show all data associated with a given tariff line including tariff rates, tariff quota, tariff safeguard, seasonality and product specific rule, as appropriate.

  • Products Rules Of Origin - Example of how to build a Rules of Origin guidance questionnaire for a given tariff line.

  • Services Hierarchy Browser - Example of how to use and implement a hierarchy view of service schedule categories.

  • Services Data Viewer - Example of how to show all data associated with a given service sector.
